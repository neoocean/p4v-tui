디프 · 히스토리

변경을 비교하고 히스토리를 파고듭니다. 통합·나란히 디프, 임의 디프, 블레임, 리비전 그래프, 타임랩스.

디프

  • 제출된 CL 통합 디프 — Submitted 에서 Ctrl+D.
  • 나란히 보기(side-by-side) — 메뉴에서. 모든 임의 디프 결과에 재사용됩니다.
  • Diff Against Have — 워크스페이스 맥락 메뉴(#have vs 작업본).
  • 임의 디프 — Ctrl+Shift+D: 두 파일 · 두 리비전(#A vs #B) · 두 폴더(/... 쌍 피커) · 두 CL(@A vs @B).

히스토리

History 패널은 파일 리프에 커서를 올리면 파일 히스토리(filelog), 디렉토리면 폴더 히스토리(changes -L)를 자동 로드합니다. Ctrl+T 로도 엽니다. 폴더일 땐 스키마가 바뀝니다(Rev·Action 열을 뺌) ➕.

Annotate · 리비전 그래프 · 타임랩스

파일 맥락 메뉴에서 세 가지 히스토리 뷰를 엽니다.

  • Annotate / 블레임 — 각 라인을 마지막으로 바꾼 CL·사용자·날짜(annotate -i -c).
  • Revision Graph — 통합(filelog -i -l)을 따라간 텍스트 모드 히스토리.
  • Time-lapse View — ,/. 로 이전/다음 리비전을 걸어보고, ←/→ 로 본문 가로 스크롤.
터미널이라 진짜 픽셀 이미지·확대형 그래프는 없지만, 텍스트 모드 등가물로 같은 정보를 담습니다.
