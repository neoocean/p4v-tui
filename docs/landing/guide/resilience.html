연결 회복력

p4v-tui 의 존재 이유. 링크가 흔들려도 작업이 살아남는 방법을 자세히 봅니다.

자동 재연결

모든 p4 호출은 _run_resilient 를 지납니다 — 드롭 시 지수 백오프 (1s→30s)로 재연결하고, 재시도 대기 중 락을 놓아 다른 명령이 사이에 끼어들 수 있습니다(논블로킹). 중간 재연결 동안 ConnectionBar 에 ⟳ Reconnecting… (attempt N/max) 가 뜹니다.

청크 · 재개 가능한 동기화

  • 청크 동기화 — 파일 단위 완료를 ~/.p4v-tui/sync-state/{hash}.json 에 기록.
  • 다음 실행 재개 피커 — 중단된 작업을 개별로 재개/버리기/건너뛰기.
  • 청크 revert / reconcile / clean / force-sync — 한 청크씩, 대화형 우선순위가 사이에 끼어듦.
  • Lost-ack 서브밋 멱등성 — "no such pending CL" 시 changes 로 실제 커밋 여부 확인.
  • 종료 시 무손상 정리 — 실행 중 청크는 마치고, 대기 청크만 취소.
  • 청크 전략 설정 — [chunking](개수/크기/단일/서브디렉토리) + 잡별 오버라이드.

운영 가시성

커맨드 모니터(F2) — 부모/자식 명령 트리와 명령별 ETA. 청크 잡이 어떻게 분해되는지 보여줍니다. (트리에 포커스가 있을 땐 F2 가 퀵 리네임이므로, 표·로그나 푸터의 f2 로 엽니다.)

로그 패널 — 모든 p4 호출·청크 잡·오류를 타임스탬프와 함께 쌓는 스크롤 가능한 꼬리. 바닥에서 자동 팔로우. 항목을 Enter 하면 ±8 줄 맥락 + 전체 트레이스백 뷰어. 예외는 크래시 대신 로그로 라우팅되고 ~/.p4v-tui/last-error.log 에 남습니다.

매크로(Ctrl+Shift+M) — [[macro]] 로 정의한 p4/sync/notify 스텝 체인을 fail-fast 로 실행. (로컬 명령을 실행하므로 신뢰할 수 있는 설정만.)
